Lecture 5

Categories

  • Things that have a common label
  • Who creates categories? Everyone - you, culture and society, institutions, computers
  • Institutional categories - explicitly defined for a purpose, created through deliberation and consensus
    • Example: US Code 61 - Gross income defined
  • Computationally created categories - When there’s shit tons of information
    • Example: Histograms, Supervised and Unsupervised ML
  • Principles for intentionally creating categories
    • Enumeration - Car Brands
      • Simplest way to define a category is by enumerating it’s members. Meaning of a category is not a property
    • Single Property - Climbable things
      • Use only the values of any single property
      • example: color, size, shape, nature
    • Multiple properties - expensive and brittle things
      • When items being categorised can be categorised using separable properties or by combining them
    • Goal based - things you can take on a camping trip
      • Ad-hoc and nameless usually
      • few discernable properties in common
    • Theory based - definition of a triangle

Classical view of categories (OUTDATED now, valid 2000yrs ago)

  • Categories are defined by a list of properties shared by all elements in a category
  • Membership is binary - in or out!
  • Membership is defined by rules, every member is equal

Categories, Wittgenstein

  • PROTOTYPICAL
  • Fuzzy, no list of rules and properties
  • Things just belong to a category and categories are defined by the members in them
  • Typicality - some things are more typical examples of a category than others

Lecture 4

Structuring information

  • Why do we describe something? Bird example - The labels depend on the purpose.
    • Birdwatching vs Image Classification AI - You’d have different labels for both
  • How do we divide up the world - Categorisation
  • Process of structuring information
    • Start with something that needs to be organised > Decide what will be retained and what will be ignored > Develop descriptions for what is being retained > Iterated on those descriptions, comparing them to evaluation criteria > Assign descriptions to the things that are being organised > XML, JSON etc.

What is Metadata?

  • Data that describes other data/information
  • Example:
    • Information: blog post
    • Metadata: language, length, author, date, publisher, keywords, sentiment
  • Metadata if for organising collections of information
    • Also for:
    • To make representation of information consistent
    • To make search and retrieval easier
    • To help exchange information
    • Support auditing
